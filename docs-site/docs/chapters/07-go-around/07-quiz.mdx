<QuizQuestion
  question="Which of these is a good refactoring prompt for an AI tool?"
  options={[
    "\"Make this code better\"",
    "\"Clean up this file\"",
    "\"Extract the date formatting logic into a shared helper function in lib/utils/date_helpers.dart\"",
    "\"Refactor this to be more maintainable\""
  ]}
  correctIndex={2}
  explanation="A good refactoring prompt names the specific transformation (extract), the specific code (date formatting logic), and the specific destination (lib/utils/date_helpers.dart). The other prompts are vague — they describe a feeling ('better', 'clean', 'maintainable') rather than a concrete action. AI cannot optimise for feelings; it can execute named transformations."

<QuizQuestion
  question="What is the 'explain then refactor' technique?"
  options={[
    "Ask the AI to explain what refactoring is, then do it yourself manually",
    "First ask the AI to explain the current code structure, verify its understanding, then ask for a specific transformation",
    "Write comments explaining every line, then ask the AI to simplify the code",
    "Ask the AI to refactor first, then ask it to explain what it changed"
  ]}
  correctIndex={1}
  explanation="The two-step technique is: (1) ask the AI to explain the code — this verifies it understands the structure before changing it; (2) ask for a specific, named transformation. If the AI's explanation is wrong in Step 1, you know not to trust it with Step 2. This catches misunderstandings before they become bugs."
/>

<QuizQuestion
  question="What is the 'Lost in the Clouds' anti-pattern?"
  options={[
    "Using AI to refactor code that is stored in cloud services",
    "Making vague refactoring requests that produce multiple rounds of unrecognisable changes, leaving you with code nobody fully understands",
    "Refactoring too many files at once in a single commit",
    "Using cloud-based AI tools instead of local ones for refactoring"
  ]}
  correctIndex={1}
  explanation="Lost in the Clouds happens when vague prompts ('refactor this', 'make it better') give the AI permission to change anything. After multiple rounds of undirected changes, the code is unrecognisable, the diff is larger than expected, and nobody fully understands what the code does. The fix: revert, identify ONE specific transformation, prompt for that only."
/>

<QuizQuestion
  question="When should you perform a 'go-around' during refactoring?"
